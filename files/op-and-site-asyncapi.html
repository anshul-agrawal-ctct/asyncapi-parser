Op & Site 0.0.1

API between Op & Site in the reference Architecture. It is designed to allow a machine control interface to run atop different positioning solutions from different suppliers

Servers

  • MQTT://server.url/ MQTT 5.0 RA-API

Operations

onOperatorNotificationReceived

Receive operatorNotifications

Description:
Channel that can be used by any software component to publish messages intended to be displayed to operator, usually for the operator to perfom an action.

Accepts the following message:
  • SchemaoperatorNotifications
    Schema Formatfbs
    OperatorNotification
    Field Name Type
    uuid string
    thread_id uint8
    level Level
    message_title string
    message_text string
    url string
controlsStateVisibility

Send controlsStateVisibility

Description:
Channel used to post the current visibility of the autos state in the operator display. Machine positioning shall disable the auto state if it is not visible to operator. O&S should change this value when a user obscures/leaves the screen that reports the current state of autos.

Accepts the following message:
  • SchemacontrolsStateVisibility
    Schema Formatfbs
    ControlStateVisibility
    Field Name Type
    visibility Visibility
localeSettings

Send localeSettings

Description:
Channel used to convey the selected locale settings (language and units) to other components for their own user interfaces and messages sent to operator.

Accepts the following message:
  • SchemalocaleSettings
    Schema Formatfbs
    Locale
    Field Name Type
    language_code string
    length_unit LengthUnit
    angle_unit AngleUnit
report

Send report

Description:
Channel used to report current status of O&S component.

Accepts the following message:
  • Schemareport
    Schema Formatfbs
    OpAndSiteReport
    Field Name Type
    uuid string
    name string
    manufacturer_name string
    manufacturer_code uint
    svg_icon string
    operator_ui_base_url string
    technician_ui_base_url string
    status Status
